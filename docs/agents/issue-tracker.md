# Issue tracker: Local Markdown

Issues live under `docs/issues/`; PRDs live under `docs/prd/`.

## Conventions

- PRDs use `docs/prd/<NNNN>-<slug>.md`.
- Implementation issues use `docs/issues/<NNNN>-<slug>.md`.
- Each issue links to its source PRD when applicable.
- Triage state is recorded as a `Status:` line near the top.
- Comments are appended under a `## Comments` heading.

## Publishing and fetching

When a skill publishes a PRD or issue, create the corresponding file under
`docs/prd/` or `docs/issues/`. When fetching work, read the referenced local
Markdown file directly.
