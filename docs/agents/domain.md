# Domain Docs

This is a single-context repository.

Before exploring or changing domain behavior:

1. Read root `CONTEXT.md`.
2. Read relevant ADRs under `docs/adr/`.
3. Use the glossary's canonical vocabulary.
4. Surface conflicts with accepted ADRs rather than silently overriding them.
5. If a required domain term is missing, note it for `grill-with-docs`.
